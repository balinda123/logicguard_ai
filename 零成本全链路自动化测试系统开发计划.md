# 🚀 LogicGuard AI：零成本全链路自动化测试系统开发计划（优化版）

## 1. 项目概览 (Architecture Overview)

### 1.1 核心目标

- **用户体验**：实现"一键安装，开箱即用"的测试工具
- **技术架构**：采用"本地执行 + 远程推理"的分布式架构
- **成本控制**：利用现有硬件资源，实现零成本测试解决方案

### 1.2 系统架构

- **算力端 (家里)**：华硕天选 6 Pro，运行 Ollama (LLM) 和 PocketBase (后端数据库)
- **执行端 (公司)**：任何办公电脑，运行 Tauri 桌面应用
- **通信桥梁**：Tailscale Funnel (内网穿透)，确保公司电脑能安全访问家里的 AI

## 2. 环境准备 (Home & Office Setup)

### 2.1 家里电脑：搭建"AI 算力中心"

- **安装 Ollama**：
  - 访问 [ollama.com](https://ollama.com) 下载安装
  - 环境变量设置：添加 `OLLAMA_HOST=0.0.0.0`（允许非本地连接）
  - 下载模型：`ollama run deepseek-v3` (逻辑推理) 或 `ollama run qwen2.5-coder:7b` (代码生成)
- **部署 PocketBase (轻量后端)**：
  - 下载 PocketBase 压缩包，解压
  - 运行：`./pocketbase serve`
  - 访问 <http://localhost:8090/_/> 创建管理员账号，并创建 `test_results` 集合
- **开启 Tailscale Funnel (内网穿透)**：
  - 安装 Tailscale
  - 访问 Tailscale 管理控制台，开启 Funnel 功能并选择 "Allow public access"
  - **关闭方法：** 如果不需要用了，在家里电脑输入：

    PowerShell
    ```
    tailscale funnel --off 11434
    ```
  - 记录生成的域名：如 <https://node.tail53aa35.ts.net/>
  - 配置说明：开启公开访问后，任何互联网用户都可以通过该域名访问服务，无需安装 Tailscale
- 要在一台电脑上同时暴露两个服务，你有两种最先进的实现方式：
  #### 方案 A：路径转发 (Path-based Routing) —— 推荐
  你可以告诉 Tailscale，根据网址后面的“小尾巴”来决定去哪：
  - 访问 `https://your-node.ts.net/`-> 转到 **11434** (Ollama)
  - 访问 `https://your-node.ts.net:8443` -> 转到 **8090** (PocketBase)
    **操作命令：**
    PowerShell
  ```
  #重置
  tailscale funnel reset
  # 以后访问 https://你的域名 就能连上 AI
  tailscale funnel --bg 11434

  # 以后访问 https://你的域名:8443 就能连上数据库
  tailscale funnel --bg --https 8443 8090
  ```

### 2.2 公司电脑：简化部署

- **无依赖安装**：用户只需下载并安装 `LogicGuardAI.exe`
- **自动配置**：首次启动时通过安装向导完成配置

## 3. 开发阶段 (Development Phases)

### 3.1 第一阶段：Tauri 桌面外壳搭建

- **技术栈**：Tauri 2.0 + React + Vite + Tailwind CSS
- **核心功能**：
  - 安装向导：引导用户完成初始配置
  - 主界面：侧边栏（任务列表、设置、报告）、主区域（测试执行面板）
  - 配置管理：自动保存和加载用户配置
  - 状态存储：使用 `localStorage` 管理登录状态

### 3.2 第二阶段：Playwright 自动化引擎

- **内置浏览器**：打包 Chromium 到应用中，无需用户单独安装
- **SSO 穿透**：使用 `launchPersistentContext` 指向用户 Chrome Profile
- **视觉反馈**：点击操作前显示红点动画
- **错误处理**：完善的异常捕获和重试机制

### 3.3 第三阶段：AI 意图解析

- **接口封装**：抽象 Ollama API 调用，处理网络异常
- **智能解析**：根据用户输入和页面截图，生成测试步骤
- **执行闭环**：解析 AI 返回结果，驱动浏览器执行操作
- **自动修复**：遇到错误时尝试自动调整策略

### 3.4 第四阶段：多用户数据同步与认证

- **应用内认证系统**：
  - 基于 PocketBase 实现用户注册、登录功能
  - 支持邮箱/密码登录和第三方 OAuth 登录
  - 实现 JWT token 认证机制
  - 应用启动时自动进行身份验证
- **访问控制**：
  - 基于角色的权限管理（管理员、普通用户）
  - 限制每个用户的 API 调用频率
  - 记录用户操作审计日志
- **结果管理**：自动生成和上传测试报告
- **版本控制**：内置自动更新功能

### 3.5 第五阶段：用户体验优化

- **测试模板**：提供常用测试场景模板
- **操作指引**：内置使用教程和帮助文档
- **性能优化**：减少资源占用，提高响应速度
- **可视化**：实时显示测试执行状态和进度

## 4. 技术实现要点

### 4.1 依赖管理

- **静态链接**：将所有依赖打包到 exe 中
- **自动检测**：检查并自动配置必要的系统组件
- **隔离环境**：使用 Tauri 的安全沙箱

### 4.2 网络管理

- **自动连接**：智能检测和连接到家里的服务器
- **故障恢复**：网络中断后自动重连
- **连接监控**：实时显示网络状态
- **备用方案**：提供离线模式（有限功能）

### 4.3 安全与隐私

- **数据脱敏**：自动屏蔽敏感信息
- **加密传输**：所有数据传输使用 HTTPS
- **认证安全**：
  - JWT token 加密存储
  - 密码哈希处理
  - 定期 token 刷新机制
- **权限控制**：最小权限原则
- **访问限制**：API 调用频率限制，防止滥用
- **审计日志**：记录关键操作和登录行为

### 4.4 部署与更新

- **安装包**：创建标准 Windows 安装程序
- **自动更新**：内置更新检测和安装
- **版本管理**：语义化版本控制
- **回滚机制**：更新失败时自动回滚
